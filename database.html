Database

TrainingRoutine utiliza Room como base de datos local. Esto permite el almacenamiento persistente de entidades como sesiones, entrenamientos y jugadores.

📁 Entidades

  • PersonaEntity: representa un jugador (nombre, posición, avatar)
  • EntrenamientoEntity: tipo de entrenamiento, descripción, fecha
  • SesionEntity: sesión de entrenamiento, lista de jugadores, estado, ubicación

🔧 Conversores

Se utilizan @TypeConverters en Converters.kt para almacenar listas de objetos como JSON en Room.

🧠 DAO Interfaces

Los DAOs permiten operaciones CRUD de forma sencilla:

@Dao
interface SesionDao {
    @Query("SELECT * FROM sesionentity")
    fun getAllSesiones(): LiveData>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertSesion(sesion: SesionEntity)
}

🏛️ Base de datos

La clase TrainingRoutineDatabase.kt proporciona la instancia Singleton de Room y sus DAOs.
